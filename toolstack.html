How They Connect

Key relationships between tools that aren’t obvious from the layer diagram.

◆ MCPs Feed the Agent

  • Perplexity gives live web search so the agent isn’t trapped in its training data
  • Context7 gives live docs so tool integrations stay current
  • X MCP gives social signals for market-aware agents (outreach, content)
  • Orgo MCP lets agents manage their own infrastructure (self-healing)

◆ Composio Is the Multiplier

  • One connector install unlocks 10+ app integrations per agent
  • Customer YAML specifies which apps each agent gets (composio_apps)
  • Gmail, Trello, Notion, etc. all run through Composio’s action layer
  • Agent Mail handles email identity separately (not through Composio)

◆ Obsidian Is Institutional Memory

  • agents.mmd tells the agent where to find its Obsidian vault
  • Agent reads from vault at task start, writes discoveries back
  • Granola meeting notes flow into the vault via Composio
  • Knowledge compounds over time — agents get smarter per customer

◆ Model Tier Strategy

  • GPT 5.5 primary handles quality-critical tool-calling for both tiers
  • Opus 4.7 fallback (Hermes only) for complex multi-step reasoning
  • GLM 5.1 light does cheap high-volume routine work
  • Kimmy light fallback (OpenClaw) is runner-up when GLM is overloaded

◆ The Operator Loop

  • Bob uses Claude Code / Codex + MCPs to build and configure agents
  • Telegram meta-agent gives Bob eyes on all 27+ VMs from one channel
  • Watchdog catches failures before customers notice (the moat)
  • Email alerts fire alongside Telegram for redundancy

◆ Hermes vs OpenClaw

  • Same provisioning code path — tier is read from stack YAML
  • Hermes adds Opus 4.7 fallback, mandatory second brain, all MCPs
  • OpenClaw ships lighter: 2 vCPU/4GB vs 4 vCPU/8GB, optional MCPs
  • Mixed-tier supported — one customer can have both Hermes and OpenClaw agents

The agents.mmd Pattern

Each agent boots with a context file that ties all the tools together. The .mmd file is a Markdown briefing document loaded into the agent’s working memory. It defines four things:

1
Who

Customer name, vertical, contacts

2
What

Agent role and responsibilities

3
Where

Obsidian paths, Composio apps, MCP configs

4
How

Escalation rules, quality standards

# agents.mmd — loaded into working memory at boot ## Identity You are outreach-agent for Acme Marketing LLC. Your role: cold outbound + reply triage. ## Second Brain Check your Obsidian vault at /data/obsidian/ for: - Customer personas (personas/) - Previous outreach templates (templates/) - Meeting notes from Granola (meetings/) ## Tools - Gmail via Composio for sending/receiving - Trello via Composio for lead tracking - Perplexity MCP for prospect research - Context7 MCP for API documentation ## Escalation If confidence < 70% on any reply, draft but do not send. Flag for human review via Telegram.
