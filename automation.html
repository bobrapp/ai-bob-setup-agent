What is this?

This is the internal automation system that powers the AIGovOps Foundation's daily operations. It's not a product we sell — it's the system we use ourselves to prove that two co-founders can operate at the scale of a full team.

Think of it like having a smart executive assistant, a community manager, a researcher, a content writer, and a project coordinator — all working 24/7, all checking with you before doing anything important.

How it works

1. Agents draft, you decide

Every consequential action — sending an email, posting to the community, publishing a newsletter — goes through the Approval Queue. You see it in Telegram with approve / reject / edit buttons. Nothing happens without your tap.

2. Everything is logged

Every agent action is recorded in an append-only audit log (logs/audit.jsonl). Who did what, when, which model, what happened. This is the AIGovOps provenance rule in action — we practice what we preach.

3. Agents watch themselves

If any agent's failure rate exceeds 10% in a 24-hour window, the system automatically suspends it and alerts you. You can also manually suspend or resume any agent via Telegram commands.

4. Dry-run mode for safety

Set dry_run: true in your config and the entire system logs what it would do without making any external API calls. Perfect for testing before going live.

Step-by-step setup

You don't need to be technical to understand the setup — but you will need someone comfortable with the command line to run the install commands.

Step 1: Choose your services

The system integrates with tools you likely already use. Here's what you need:

Start small.

You don't need all services on day one. Start with Telegram + Asana + Make.com (the core), then add Circle.so, Granola, and Perplexity as you need them.

Step 2: Install the system

git clone https://github.com/bobrapp/ai-bob-setup-agent.git
cd ai-bob-setup-agent
make install-foundation

Step 3: Configure your credentials

cp config/personal-foundation/config.example.yaml config/personal-foundation/config.yaml
# Open config.yaml and fill in your API keys and chat IDs

Step 4: Verify everything works

make doctor-foundation

This checks that your config is valid and all required services are reachable.

Step 5: Test with dry-run mode

Set dry_run: true in your config. The system will log everything it would do without actually calling any external APIs. Review the logs to make sure it behaves as expected.

Step 6: Go live

Set dry_run: false and start the system:

make run-foundation

You'll start receiving Approval Queue items in your Telegram channel. Approve, reject, or edit as they come in.

Common questions

Can the agents send emails or post without my permission?

No. Every external communication passes through the Approval Queue. You must explicitly tap "Approve" in Telegram before anything goes out.

What if an agent makes a mistake?

You can reject any action with a reason, and the agent will revise. If an agent fails too often (>10% failure rate), it's automatically suspended until you investigate.

Do I need to be technical to use this?

You need someone technical for the initial setup (30 minutes). After that, your daily interaction is just Telegram — approve, reject, or edit. No command line needed.

What about Marblism or Sintra?

We evaluated both. They're great for some use cases, but neither exposes a per-action audit log — which is a hard requirement for the AIGovOps provenance rule. If they add that feature, we'll integrate them.

How much does it cost to run?

The software is free (open source). Your costs are the services you choose: Telegram (free) + Make.com ($12/mo) + whatever tools you already pay for. Most people are already paying for email and project management.

Is my data safe?

The audit log never stores email bodies, post content, or personal information. Only metadata (subject lines, member IDs, action summaries) is logged. Your config file with API keys is gitignored and never committed.
